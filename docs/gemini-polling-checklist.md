# Gemini Polling Checklist (Hui)

이 문서는 Gemini CLI(휘)가 `tap-comms` 환경에서 안정적으로 폴링(Polling)을 운영하기 위해 준수해야 할 체크리스트이다.

## 1. 초기화 단계 (Initialization)
- [ ] **에이전트 이름 설정**: 세션 시작 직후 `tap_set_name(name: "휘")`를 호출하여 수신 대기 상태를 확정했는가?
- [ ] **인박스 경로 확인**: `D:/HUA/hua-comms/inbox` 경로가 정확히 마운트되어 있는가?
- [ ] **BOM 스트립 활성**: Windows 환경에서 작성된 Markdown 파일의 BOM 제거 로직이 작동하는가?

## 2. 폴링 운영 (Polling Execution)
- [ ] **폴링 주기**: 5~10초 간격으로 `tap_list_unread`를 호출하고 있는가?
- [ ] **자가 수신 방지**: `from === AGENT_NAME`인 메시지를 무시하여 무한 루프를 방지했는가?
- [ ] **중복 처리 방지**: 이미 읽은 메시지의 마커(Marker) ID를 캐싱하여 중복 알림을 막고 있는가?

## 3. 예외 및 복구 (Exception & Recovery)
- [ ] **연결 유실 대응**: MCP 서버(`tap-comms`) 연결이 끊겼을 때 재연결 로직이 작동하는가?
- [ ] **메시지 누락 체크**: 폴링 간격 사이에 생성된 메시지가 누락되지 않도록 인박스 히스토리를 훑는가?
- [ ] **Windows 파일 잠금**: 파일 읽기 시 발생하는 Race Condition(공유 위반)에 대해 재시도 로직이 있는가?

## 4. 로깅 및 알림 (Logging & Notify)
- [ ] **터미널 알림**: 새 메시지 도착 시 벨소리(`\x07`)와 함께 가독성 있는 로그가 출력되는가?
- [ ] **메시지 요약**: 전체 본문을 읽기 전 제목과 발신자를 한눈에 확인할 수 있는가?
